```bash
astro25 write-channel INPUT.cpg OUTPUT.cpg --zone N --slot N [--label TEXT] [--rx MHZ] [--tx MHZ] [--tx-pl HZ] [--save-time YYYY-MM-DDTHH:MM] [--no-save-metadata]
```

## What It Can Do

### W6SLG / XTS multizone family

Supported now:

- edit visible labels in `84c5`
- edit RX/TX frequencies in `84c7`
- update CPS-style `8b7d` save metadata
- recompute `84c5` / `84c7` tails
- recompute `8b7d` tails
- mirror updated CRC/tail values into `Strm_0`
- preserve the observed `84c5` directory quirk for the first long W6SLG label table (`tail XOR 0x0100`)

Experimental:

- TX PL updates for split-pair rows using the current “next entry flag bytes” heuristic

### Legacy single-zone XTS family

Supported now for already-initialized/programmed files:

- edit visible labels in `84c5`
- edit RX/TX frequencies in `84c7` when the row already decodes as `a_plus_pair` or `uniform`
- edit TX PL in `84c7` through the confirmed control byte
- update CPS-style `8b7d` save metadata
- recompute `84c5` / `84c7` tails
- recompute `8b7d` tails
- mirror updated CRC/tail values into `Strm_0`

Not supported yet:

- promoting the original 46-byte blank-template `84c7` into the fully initialized 47-byte programmed form

## Save Metadata Rules Now Modeled

For the validated XTS families, `8b7d` is no longer opaque.

- `8b7d` len `50`
- bytes `4..8` hold a BCD timestamp `YY MM DD HH MM`
- W6SLG multizone `8b7d` tail fits `CRC16-CCITT(decrypted[4:-2])`
- legacy single-zone `8b7d` tail fits `CRC16-CCITT(decrypted[4:-2]) XOR 0x26aa`

The writer now updates those fields directly unless `--no-save-metadata` is used.

## Directory Rules Now Modeled

The W6SLG multizone family also has one important non-literal directory rule:

- the first long `84c5` table (`record_index_word = 0`) stores its `Strm_0` CRC as `stream_tail XOR 0x0100`

That rule is now encoded in the writer, which is why the `Codex` and `AAPC` checkpoints can be reproduced exactly.

## Important Limitations

The writer is still not full CPS parity.

Main current limitations:

- the W6SLG multizone TX PL heuristic is still experimental
- RX PL / carrier-squelch mapping is not solved yet
- blank-template materialization for untouched single-zone blanks is not automated
- more complex edits like scan, zone moves, personality rewrites, or adding/removing channels are not implemented

## Bottom Line

The writer is now good enough to be useful for focused channel edits in the families we understand best:

- visible label edits
- RX/TX frequency edits
- TX PL edits for the programmed single-zone family
- experimental TX PL edits for the W6SLG multizone family

It is no longer just a decoder with ideas about writing. It now reproduces real CPS-authored channel changes closely enough that the remaining gap is mostly save-metadata rather than core channel data.
It now reproduces the current checkpoint edits exactly for the best-understood XTS families, with the remaining uncertainty concentrated in deeper channel semantics like RX PL and the multizone TX PL side-effect behavior.
